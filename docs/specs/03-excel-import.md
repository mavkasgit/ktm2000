# 03. Excel Import

## Цель

Импорт должен принять привычный Excel-план, сохранить оригинальный файл, нормализовать строки и создать или обновить позиции общего производственного плана. Excel является первым источником данных для проверки системы, но не корневой сущностью планирования.

## Поддерживаемый формат v1

Файлы Excel/табличных книг, поддерживаемые `python-calamine`:

- `.xls`;
- `.xlsx`;
- `.xlsm`;
- `.xlsb`;
- `.ods`.

Расширение проверяется по whitelist, но фактическое чтение выполняется через `python-calamine`, потому что пользователи могут передать файл с двойным или неточным расширением.

Обязательные поля после сопоставления колонок:

- `month` или `period`;
- `sku`;
- `product_name`;
- `quantity`.

Для упаковочных карт РП количество производственного плана берется из колонки `кол-во штук готовой продукции`. Колонки `Запад` и `Восток` отражают план отгрузки и не влияют на производственный план.

Рекомендуемые поля:

- `due_date`;
- `period_start`;
- `period_end`;
- `customer`;
- `priority`;
- `order_ref`.

## Шаблон сопоставления колонок

Система должна хранить `ImportTemplate`:

- `id`;
- `name`;
- `column_mapping jsonb`;
- `created_by`;
- `created_at`;

Пример mapping:

```json
{
  "month": "Месяц",
  "sku": "Артикул",
  "product_name": "Наименование",
  "quantity": "Количество",
  "due_date": "Срок готовности",
  "priority": "Приоритет"
}
```

Если в файле названия колонок совпали с шаблоном, пользователь сразу видит preview. Если нет - открывается экран ручного сопоставления.

## Этапы импорта

1. Upload.
2. Сохранить оригинальный файл в локальное файловое хранилище.
3. Посчитать SHA-256.
4. Прочитать первый лист через `python-calamine`.
5. Найти строку заголовков.
6. Применить шаблон колонок.
7. Нормализовать значения.
8. Создать `ImportBatch` со ссылкой на файл и режим импорта.
9. Создать `PlanPosition` в статусе `draft` или привязать строки к существующему производственному плану.
10. Запустить валидацию.
11. Вернуть preview с ошибками.

## Режимы импорта

Пользователь должен явно выбрать режим:

- создать новый `ProductionPlan`;
- добавить позиции в существующий `ProductionPlan`;
- заменить черновые позиции, созданные предыдущей версией этого же импорта.

Запущенные в производство позиции импорт не изменяет. Для них создается новая позиция-корректировка или предупреждение.

## Diff and restore points

Перед применением импорта система создает `PlanChangeSet`.

Дедупликация выполняется через source-поля, а не через primary key:

- `file_sha256` - отпечаток загруженного файла;
- `source_row_hash` - отпечаток нормализованной строки;
- `source_ref` - внешний номер строки/заказа, если есть;
- `external_plan_id` - стабильный внешний идентификатор позиции, если есть;
- `source_fingerprint` - расчетный ключ из SKU, периода, срока, клиента/заказа и количества.

Для каждой строки показывается действие:

- `ignore_unchanged` - такая позиция уже есть, ничего не меняем;
- `create_position` - новая позиция будет добавлена;
- `update_draft_position` - черновая позиция будет обновлена;
- `mark_possible_duplicate` - похоже на дубль, нужно решение пользователя;
- `cancel_draft_position` - черновая позиция отсутствует в новом файле и может быть отменена.

Пользователь применяет изменения только после просмотра diff.

Откат:

- если примененный `PlanChangeSet` не затронул запущенные позиции, его можно откатить целиком;
- откат удаляет или отменяет созданные черновые позиции и возвращает обновленные черновики к `before_data`;
- если позиция уже вошла в `ReleaseBatch`, откат запрещен, вместо него нужна `PlanAdjustment`.

## Валидации

Блокирующие ошибки:

- отсутствует обязательная колонка;
- SKU пустой;
- quantity пустой;
- quantity <= 0;
- невозможно распарсить дату;
- изделие по SKU не найдено;
- изделие неактивно;
- у изделия нет активного BOM;
- у изделия нет активного маршрута;
- в маршруте нет этапов;
- маршрут содержит неактивный участок.

Предупреждения:

- нет `due_date`;
- нет `priority`;
- SKU найден, но имя в Excel отличается от имени в базе;
- загружен файл с таким же SHA-256;
- в плане есть дубли SKU + due_date.

## Нормализация

Количество:

- заменить пробелы;
- заменить запятую на точку;
- преобразовать в decimal;
- запретить отрицательные значения и ноль.

Дата:

- поддержать Excel serial date;
- поддержать ISO `YYYY-MM-DD`;
- поддержать локальный формат `DD.MM.YYYY`.

Период:

- если задан месяц, хранить `period_start` как первый день месяца и `period_end` как последний день месяца;
- если задан диапазон дат, хранить фактический диапазон;
- если период не задан, разрешить черновую позицию, но блокировать запуск в производство до уточнения правил планирования.

SKU:

- trim;
- сохранить исходный регистр;
- поиск делать case-insensitive, если в базе SKU уникален без учета регистра.

Упаковочная карта РП:

- строка с `кол-во штук готовой продукции > 0` является кандидатом в позицию плана;
- итоговая позиция определяется не только артикулом, а набором параметров выхода: сырьевые артикулы, цвет, входная длина, операция, упаковка, длина после упаковки, вид отгрузки, период;
- строки `Запад` и `Восток` сохраняются в `source_payload`, но не участвуют в `source_fingerprint`;
- строки с пустыми базовыми полями могут наследовать контекст только от ближайшей предыдущей полной строки с тем же артикулом;
- строки с другим артикулом и пустым наименованием не наследуют название автоматически, а участвуют в распознавании парного профиля или получают warning.

## Парный профиль

Парный профиль - это готовый выход, который производится из двух или более сырьевых артикулов. В Excel он может быть представлен соседними строками с разными артикулами, но одинаковыми параметрами выхода: цвет, количество готовой продукции, длина после упаковки, вид отгрузки и упаковочные признаки.

Правила v1:

- парный профиль импортируется как одна позиция плана, а не как несколько независимых позиций;
- сырьевые артикулы сохраняются в `source_payload.components`;
- `PlanPosition.product_id` должен ссылаться на итоговый продукт, если он уже заведен;
- BOM итогового продукта описывает сырьевые компоненты парного профиля;
- если итоговый продукт/BOM не найден, change set создает позицию с warning `paired_profile_product_unmapped` и не считает ее готовой к утверждению.

## Повторный импорт

Если файл с таким же SHA-256 уже был загружен:

- показать предупреждение;
- разрешить создать новый черновик;
- запретить автоматическую генерацию без явного подтверждения.

## Preview результата

Экран импорта должен показывать:

- общее число строк;
- валидные строки;
- строки с ошибками;
- строки с предупреждениями;
- найденные изделия;
- неизвестные изделия;
- изделия без BOM;
- изделия без маршрута.

Позиции можно утвердить только если по ним нет блокирующих ошибок. Один производственный план может содержать одновременно утвержденные, черновые и ошибочные позиции.
