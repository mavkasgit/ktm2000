# Testing Guide — KTM-2000

## Обзор

Проект KTM-2000 использует два уровня тестирования:
1. **Интеграционные и юнит-тесты бэкенда (pytest):** Покрывают бизнес-логику бэкенда, эндпоинты, импорт Excel-файлов, планирование и бэкапы. Работают с изолированной тестовой базой данных PostgreSQL.
2. **E2E-тесты интерфейса (Playwright):** Проверяют сквозные сценарии работы пользователя в браузере. Могут запускаться в CDP-режиме (подключение к Chrome через порт 9222) или в режиме Headless.

---

## Тестирование бэкенда (pytest)

Тестирование выполняется с использованием pytest на выделенной тестовой базе данных PostgreSQL (порт `5212`).

### 1. Подготовка и запуск тестовой БД
Для тестов используется Docker-окружение, настроенное в файле `infra/compose/docker-compose.test.yml`.
Чтобы запустить тестовую базу данных, выполнить миграции и дождаться её готовности, запустите скрипт из корня:
```bash
npm run test:db:up
npm run test:db:wait
```

### 2. Запуск тестов бэкенда
Для запуска тестов обязательно перейдите в директорию `backend/`, чтобы подхватились файлы настроек `pytest.ini` и `conftest.py`:

```bash
cd backend
python -m pytest tests/ -v
```

Или используйте общую скрипт-команду из корня проекта:
```bash
npm run test:pytest
```

### 3. Полезные опции pytest
- **Параллельный запуск (в 4 воркера):** ускоряет выполнение тестов. Требуется пакет `pytest-xdist` (`pip install pytest-xdist`).
  ```bash
  cd backend
  python -m pytest tests/ -v -n 4
  ```
- **Запуск конкретного файла или теста:**
  ```bash
  cd backend
  python -m pytest tests/test_shopfloor_api.py::test_shopfloor_over_issue_rejected -v
  ```

### ⚠️ Предупреждение для Windows-систем (Redirect Warning)
На операционных системах семейства Windows стандартное перенаправление вывода `2>&1 | head -80` (или аналогичное через пайплайны) может вызывать сбои или зависания pytest в некоторых эмуляторах терминала (символ `2` считывается интерпретатором как числовой аргумент). 

Запускайте тесты напрямую без пайплайнов перенаправления, либо перенаправляйте вывод непосредственно в текстовый файл для последующего анализа:
```bash
python -m pytest tests/ -v > test_output.txt 2>&1
```

## E2E-тестирование (Playwright)

Сквозные тесты интерфейса находятся в каталоге `e2e/`.

### 1. Режимы запуска браузера
Playwright поддерживает два режима тестирования (настраивается через переменную окружения `E2E_BROWSER_MODE` в `.env`):
- `cdp` (по умолчанию) — подключается к уже открытому браузеру Chrome с удаленной отладкой.
- `headless` — запускает изолированный headless-браузер Playwright.

### 2. Использование CDP-режима (Chrome DevTools Protocol)
Перед запуском тестов убедитесь, что браузер Chrome запущен с портом удаленной отладки `9222`:
- **Windows:**
  ```cmd
  chrome.exe --remote-debugging-port=9222
  ```
- **Linux:**
  ```bash
  google-chrome --remote-debugging-port=9222
  ```

Запустите тесты с помощью Playwright CLI:
```bash
npx playwright test
```
