# Deployment — KTM-2000

## Обзор

KTM-2000 разработан для локального и серверного развертывания с использованием Docker Compose для управления контейнерами. В режиме Production все компоненты системы (Frontend, Backend, PostgreSQL) упаковываются в Docker-контейнеры.

## Сетевые режимы работы (Порты)

Проект поддерживает два параллельных режима доступа:

1. **Режим единой общей сети (порт 80 / 443)**:
   - Контейнеры подключаются к внешней сети `shared_gateway`.
   - Запросы направляются через центральный обратный прокси на порту 80/443 (используя имена хостов контейнеров, такие как `ktm2000-nginx-prod`).

2. **Автономный режим (прямой доступ)**:
   - Доступ к KTM-2000 осуществляется напрямую через порт хоста **8082** (проброшен в `docker-compose.prod.yml` на порт Nginx).
   - В системе также параллельно работают `hrms` (на порту **8081**) и `ekranchik` (на порту **8083**).

## Структура конфигурационных файлов окружения (.env)

Проект использует разные файлы окружения для разделения баз данных и портов:
- `.env.dev` — Конфигурация для локальной разработки.
- `.env.test` — Настройки тестового окружения (запуск pytest / Playwright).
- `.env.prod` — Production-настройки для развертывания на сервере или в цеху.
- `.env` — Файл в корне проекта, хранящий текущие настройки хоста и доступы к MCP-серверам (SSH/SFTP).

### Ключевые переменные в `.env` файлах:

| Переменная | Пример значения | Описание |
|------------|-----------------|-----------|
| `EXTERNAL_PORT` | `8020` | Внешний порт, на котором доступно приложение |
| `BASE_URL` | `http://localhost:8020` | Базовый URL приложения |
| `POSTGRES_USER` | `ktm2000_user` | Имя пользователя базы данных PostgreSQL |
| `POSTGRES_PASSWORD` | `secure_password` | Пароль базы данных PostgreSQL |
| `POSTGRES_DB` | `ktm2000_prod` | Название базы данных PostgreSQL |
| `POSTGRES_PORT` | `5432` | Внешний порт PostgreSQL (для подключения извне) |
| `DATABASE_URL` | `postgresql+asyncpg://...` | Строка подключения ORM к PostgreSQL |
| `SECRET_KEY` | `very-long-secret-key` | Ключ шифрования сессий и токенов |
| `ENV` | `prod` / `dev` / `test` | Тип запущенного окружения |

## Production развертывание (Docker Compose)

Для запуска production-сборки на сервере:

### Шаг 1. Настройка окружения
Убедитесь, что файл `.env.prod` в корне проекта заполнен корректными секретами, паролями баз данных и внешними портами.

### Шаг 2. Сборка и запуск контейнеров
Запустите команду сборки и поднятия контейнеров из корня проекта:
```bash
npm run prod:up
```
Эта команда автоматически соберет Docker-образы для фронтенда и бэкенда, создаст том (volume) для хранения данных Postgres и запустит все сервисы в фоновом режиме (`-d`).

### Шаг 3. Просмотр логов
Для отслеживания логов запущенного production-окружения используйте:
```bash
npm run prod:logs
```

### Шаг 4. Остановка production-окружения
Для остановки и удаления контейнеров выполните:
```bash
npm run prod:down
```

## Резервное копирование (Backups)

База данных Postgres сохраняет свои данные на постоянный том Docker (volume) или локальную папку, указанную в `POSTGRES_DATA_PATH`.
Для ручного управления бэкапами и восстановления данных в системе предусмотрены соответствующие API-эндпоинты и раздел в интерфейсе настроек приложения.
Резервные копии сохраняются в директорию, настроенную через `BACKUPS_PATH` в файле конфигурации.
